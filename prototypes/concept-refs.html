참조는 종류가 아니라 배선이 정한다 — 어떤 ref든, 원하는 씬에만

product·character·environment 다 똑같음. 공유/씬별은 고정이 아니라 "어느 image에 잇느냐"의 결과.

scene 1·2 = 인물 A
scene 3 = 인물 B — 캐릭터도 씬마다 다르게 OK.
env는 scene 2만 — 씬별.
product만 전 씬 공유.
각 image = 자기한테 이어진 ref들의 합.
그게 그 씬의 "레시피".
전부로 = 공유 일부로 = 그 씬들 다른 값 = 다른 씬 하나로 = 씬 전용
핵심: 어떤 ref 종류도 공유/씬별로 고정 아님.  참조가 어느 image들에 연결됐냐가 전부 —  → 3개 다 = 공유 · → 2개 = 그 씬들 · → 1개 = 씬 전용.  그래서 character A(1·2) / character B(3)처럼 씬마다 다른 인물도 그냥 배선으로 표현.

② 그럼 템플릿(C)은 뭘 자동화하나

템플릿 ×N = 체인 구조(script→image→clip→vo)와 "전부로 가는 공유 ref"를 자동 복제.
일부·특정 씬에만 가는 ref(character B→3, env→2 같은)는 그 인스턴스에서 유저가 이어서 오버라이드.
→ 즉 "전 씬 공통"은 공짜, "이 씬만 달라"는 유저가 클릭 한 번. 반복은 사라지고 개별성은 남는다.
