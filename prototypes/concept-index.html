overall(split) → 씬 넘버(scene#)를 키로 각 노드에 연결하는 설계

출력은 순서 있는 리스트. 각 항목이 scene# 키를 달고 나오고, 그 키가 다운스트림 체인의 정체성이 된다.

① 바인딩 — 출력 항목 #i → scene# = i 체인

overall · split
shotCount = 3

한 노드가 리스트를 뱉고, 항목마다 scene# 키가 붙는다.
설계 = scene#를 "안정 키"로.
1overall 출력 = 순서·키 있는 리스트 (scene[0..N-1])
2각 체인은 scene# 정체성을 소유 (script/image/clip/vo 모두 같은 #)
3연결 = 키 매칭 (positional 배선이 아니라 #로 라우팅)

② 왜 "키"라야 하나 — 재생성 / N 변경에서 드러남

overall · split
3 → 4 로 변경
scene #1
유지 · 제자리 업데이트
scene #2
유지 (내 오버라이드·수동편집 생존)
scene #3
유지
scene #4
새로 추가
키 매칭이라서:
· 같은 N → #키로 제자리 갱신
· 3→4 → #4만 추가, #1–3 그대로
· 줄면 없어진 #만 제거

positional·재생성 방식이면 매번 싹 지우고 다시 만들어 오버라이드가 날아감.
한 줄: overall은 scene#로 태그된 리스트를 뱉고, 각 씬 체인은 자기 scene#로 그 항목을 집는다.  연결은 "몇 번째"가 아니라 "몇 번 씬"으로 — 그래서 재생성·개수변경·개별수정이 다 안전하게 공존.
