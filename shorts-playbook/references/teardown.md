# Teardown — Original Reel → Reusable Skeleton → Re-skin

The teardown turns a reference reel into a reusable skeleton, then pours a new product into it. The whole value is in the keep/regenerate line.

## KEEP (extract faithfully — this is the reusable skeleton)

- **Hook archetype** — curiosity, disbelief, problem/solution, before-after, price-shock, etc. (Name it.)
- **Structure** — the ordered beats (e.g. title hook → product reveal → process compressed → result reveal → CTA).
- **Pacing** — cut interval, total beat count, and the compression rule (e.g. "entire process compressed into 10s, 1–1.5s cuts, no dead frames").
- **CTA mechanic** — the funnel pattern (e.g. comment keyword → DM link, soft funnel).
- **SEALCaM footage specs** — per beat: Subject, Environment, Action, Lighting, Camera, Music/Sound. This is what the imagery engine (Higgsfield) consumes downstream.

## REGENERATE (never lift from the source)

- **VO** — the source reel's VO is generic creator-voice too. Lifting it inherits its flatness. Compose fresh per `vo-rules.md` with a chosen persona.
- **Captions** — authored fresh as the *claim* for each new product's beat.

## Procedure

1. Watch / read the reference reel. Identify and NAME the hook archetype.
2. Write the ordered beat structure with timings. This is the skeleton.
3. Capture the pacing rule and CTA mechanic as explicit fields.
4. For each beat, write a SEALCaM spec describing the SHOT (not the source's specific product — the shot *type*, so it transfers).
5. **Discard the source VO entirely.** Do not store it as a template.
6. Re-skin: map the new (related) product onto each beat. Captions become the new product's claims. SEALCaM specs get the new product dropped in.
7. Hand the filled skeleton to the VO step (continuous monologue, persona, then split).

## Worked transform (abbreviated)

**Source skeleton (from a 20-min ice-cream-maker reel):**
- Hook: curiosity ("you don't need an ice cream shop")
- Beats: title hook → product reveal (close-up) → add base (action) → add mix-ins → assemble/one-button → wait (time card) → result reveal → CTA
- Pacing: full process compressed to ~10s, 1–1.5s cuts
- CTA: comment keyword → DM (soft funnel)

**Re-skinned onto a different single-serve kitchen gadget:** keep all 8 beats, the pacing, and the CTA. Swap the product in the SEALCaM specs and write new captions for the new claims. Then regenerate VO natively for the chosen persona. The structure is identical; only the skin changed.

## Note on working language

The teardown may be drafted in any language that's fastest to think in — the skeleton is just data and travels fine. But the VO and captions are authored natively in US English in a later step, never translated from the working notes.
