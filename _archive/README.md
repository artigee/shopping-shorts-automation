## 0. 목적 · 결과물 (확정)
로컬에서 도는 **개인용 웹 도구**로 이 워크플로우를 빠르게 한 바퀴 돌린다:
> **find → 스크립트·로직 재구성 → 제작 리소스 찾기 → 업로드**

- **채널**: TikTok(비즈니스) + Instagram — **2채널**
- **니치**: **K-뷰티** + **일상용품** (트렌드/바이럴 기반)
- **핵심 기능 4**:
  1. 바이럴 **제품** 발굴
  2. **왜** 바이럴인지 (이유 분석)
  3. 제품 **리소스** 찾기 + **아마존 구매가능 검증**(게이트)
  4. **링크트리** 링크 허브 (활용도 미정)
- **수익화**: 아마존 어소시에이트 (+ 링크트리 허브)

## ★ 제작 컨셉 (확정) — "AI 생성"이 아니라 "골라서 조립"
1. **구조·훅** = 발굴에서 고른 **인기 릴스 구조를 템플릿**으로 상속.
2. **footage** = **제품 기준**으로 RED(샤오홍슈) 등에서 검색 → **후보 클립 큐레이션** → 유저 선택.
3. [구조 템플릿] + [선택 footage] → 조립.
4. **AI 생성(Higgsfield)** = footage 없을 때 **보조**로만.
> RED footage 소싱은 방법만 확정, **구현은 후순위**. 저작권/중복은 믹스/레퍼런스로 회피.

## 발굴 철학
조회수는 거짓말함 → 진짜 신호는 **판매/구매의도**.
- 지금: 무료 스크래퍼 + **댓글=구매의도** 대리지표 (검증됨).
- 나중: 틱톡샵 판매검증 도구(EchoTik/FastMoss/Kalodata)를 발굴 레이어로. *틱톡 채널이 있어 생태계가 붙음.*
- 무기: **크로스마켓 차익거래** — 해외(중국 RED 등) 막 뜨는 제품을 포화 전에.

## 설계 원칙 (오늘 페인포인트 = 요구사항)
1. **확인부터 받고 매 스텝.**
2. 프롬프트/선택 **항상 노출 + 인라인 편집**.
3. 결과 **즉시 미리보기 + 주석 피드백** → 그 부분만 재처리.
4. **비싼 영상 생성 최소화** — footage 소싱 우선, AI는 fallback.
5. 제품 이미지 = **마스터 레퍼런스 고정**.
6. 소스·네트워크·지역·니치 = **설정/플러그인**으로 분리.

## 빌드 백로그 (순서 · 매 스텝 확인)

### STEP A — 발굴 콕핏  ✅ 프로토타입 있음 (step1_discovery_cockpit.html)
바이럴 쇼핑 릴스 점수화·분석·선택 = **구조 템플릿 확보**.
- 채널/니치/소스 설정, **점수 가중치 슬라이더**, 자동 분류, 데이터 불러오기.
- TODO: TikTok 소스, K뷰티/일상용품 니치별 해시태그셋, 스케줄 diff(velocity), 제품단위 집계.

### STEP B — 제품 식별 + 아마존 검증 게이트
선택 릴스 → 제품 확정(스토어프론트/캡션/역이미지) → **아마존에서 구매가능 확인**(없으면 탈락) → ASIN·정보·이미지.

### STEP C — Footage 피커 (RED 소싱)  ※ 방법 확정, **후순위 구현**
제품 기준 RED 검색 → 후보 클립 갤러리 → 선택. 선결: RED 접근/검색·저작권 정책.

### STEP D — 조립
[구조 템플릿] + [선택 footage] → 컷·자막·VO(TTS)·BGM·CTA → CapCut export. footage 부족분은 AI 보조.

### STEP E — 멀티 플랫폼 릴리즈
TikTok + Instagram 자동 업로드 + 캡션·해시태그·CTA(댓글→DM) + 링크트리/자동DM 어필리에이트.
