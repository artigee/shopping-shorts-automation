# 쇼핑 쇼츠 자동화 — 프로젝트 기획안 (v1)

작성 2026-06-24 · 오늘 수동으로 한 바퀴 돌린 워크플로우를 하나의 웹앱으로 발전시키는 기획.

---

## 1. 한 줄 비전
바이럴 쇼핑 릴스를 **발굴 → 제품 리서치 → 스크립트/씬 → 이미지 → 영상 → 편집 → 멀티 플랫폼 릴리즈**까지, 사람이 결정만 하면 나머지는 자동으로 흘러가는 **반자동 제작 파이프라인 웹앱**.

---

## 2. 오늘 검증한 파이프라인 (수동)
1. **자료 수집** — 인스타 #amazonfinds 등에서 바이럴 릴스 발굴(조회수·좋아요·댓글 점수화) → 검증된 후보.
2. **스크립트 생성** — 훅 선택(훅뱅크), 문제→해결→CTA 구조, 셀링포인트 정리.
3. **씬(스토리보드) 구성** — 컷별 화면·자막·VO·필요 에셋 정의.
4. **이미지 생성** — 제품 레퍼런스 고정 + 씬별 ON/OFF 키프레임.
5. **영상 생성** — 키프레임으로 off→on 전환/모션 클립.
→ (오늘 못 한 단계: 편집 자동화, 멀티 플랫폼 릴리즈)

---

## 3. 오늘 배운 것 = 기능 요구사항 (중요)
오늘 시간·크레딧이 샌 지점이 곧 앱이 풀어야 할 문제다.

- **프롬프트가 안 보이면 헛돈다.** → 앱은 생성에 쓰는 **프롬프트를 항상 노출 + 인라인 편집** 가능해야 함. (사람이 검열 후 실행)
- **결과를 바로 못 보면 핑퐁.** → 생성 결과를 **즉시 미리보기 + 주석(화살표·동그라미) 피드백** 가능하게.
- **영상 재생성이 제일 비쌈(8.75 vs 이미지 2 크레딧).** → 반복은 **싼 모델/저해상도(1k)로 이미지 단계에서** 끝내고, 영상은 **확정된 키프레임으로 1번만**.
- **"부드러운 점등" 등 미세 모션은 AI 영상이 약함.** → 그런 효과는 **편집에서 디졸브**로 처리(앱이 OFF/ON 프레임을 자동 디졸브 트랙으로 넘김).
- **제품 일관성** = 아마존 제품 이미지를 **마스터 레퍼런스로 고정**하고 모든 컷에 물림.
- **ON/OFF 키프레임 방식이 정확**하다 — 이미지 단계에서 "끝(ON)" 먼저 확정 → "시작(OFF)" 파생 → 영상은 둘 사이 보간.

## 4. 앱 구조 (스테이지별)

### Stage 1 — 발굴 & 자료 수집
- 소스(인스타/틱톡/유튜브) + 해시태그/계정 입력 → 참여지표로 자동 랭킹.
- 카드 갤러리에서 1~2개 **선택**. (오늘 만든 갤러리 HTML이 프로토타입)
- 산출물: 선택 릴스 + 지표.

### Stage 2 — 제품 리서치 & 데이터 준비
- 선택 릴스 → 작성자 스토어프론트/캡션/역이미지로 **제품 식별**(ASIN).
- 자동 수집: 제품 정보(스펙·가격·치수·이미지), **인기 이유**, 카테고리, 경쟁 릴스.
- **훅 적용**: 훅뱅크에서 제품 카테고리에 맞는 훅 자동 추천.
- 산출물: "제품 데이터시트"(제작에 필요한 모든 사실 + 훅 후보).

### Stage 3 — 스크립트 & 씬 구성
- 데이터시트 기반 스크립트 자동 초안(훅→문제→해결→CTA) → 사람이 편집.
- 씬(컷)별: 화면·자막·VO·필요 에셋 자동 분해 → 스토리보드 테이블.

### Stage 4 — 이미지 생성 (반복의 핵심)
- 컷별 **ON(끝) / OFF(시작) 키프레임** 생성.
- **프롬프트 패널**: 항상 표시 + 편집 + "그대로 실행".
- **싼 반복 모드**: 저해상도/저가 모델로 빠르게 굴리고, 확정되면 고해상.
- 마스터 제품 레퍼런스 자동 첨부.
- 결과 위에 **주석 피드백** → 그 부분만 재생성.

### Stage 5 — 영상 생성
- 확정된 OFF→ON 키프레임으로 전환 클립 1회 생성(크레딧 절약).
- 모델/길이/모션 프리셋(전환·토글·핸드 등).

### Stage 6 — 편집 핸드오프 (CapCut 자동)
- 클립·이미지·자막·VO·BGM을 **타임라인 순서대로 자동 구성**해 CapCut(또는 드래프트 포맷)으로 export.
- **VO**: 스크립트 → TTS(ElevenLabs/Typecast) 자동.
- **Sound/BGM**: 무드별 기본 트랙.
- **점등 등 미세효과**: OFF→ON 프레임을 디졸브 트랙으로 자동 배치.

### Stage 7 — 멀티 플랫폼 릴리즈 (미구현, 연동 예정)
- 완성본 → 인스타 릴스 / 틱톡 / 유튜브 쇼츠 자동 업로드 + 캡션/해시태그/CTA(댓글→DM) 세팅.
- 자동 DM(ManyChat류)에 어필리에이트 링크 연결.

---

## 5. 기술 스택 / 아키텍처 (제안)
- **프론트**: 웹앱(React) — 스테이지별 화면 + 프롬프트 편집 패널 + 미리보기/주석.
- **생성 엔진**: Higgsfield(이미지 nano_banana / 영상 kling) API, ElevenLabs/Typecast TTS.
- **데이터 수집**: 인스타 web_info API(로그인 세션) 등 — ToS·차단 주의(반자동 권장).
- **상태/저장**: 프로젝트 단위로 에셋·프롬프트·버전 관리(에셋 보드가 이 역할의 프로토타입).
- **편집 export**: CapCut 드래프트(JSON) 자동 생성 → 사용자가 열어 최종 다듬기.
- **릴리즈**: 각 플랫폼 API/Chrome 자동화.

---

## 6. 로드맵 (단계별)
- **MVP(1차)**: Stage 1~5 + 에셋 보드 정리 (오늘 한 것의 반자동화). 프롬프트 편집·싼 반복이 핵심.
- **2차**: Stage 6 CapCut 자동 핸드오프 + TTS/BGM 기본값.
- **3차**: Stage 7 멀티 플랫폼 릴리즈 + 자동 DM 어필리에이트.
- **4차**: Stage 2 리서치 자동화 고도화(제품 데이터시트 자동 완성), 스케줄링(매일 자동 발굴).
